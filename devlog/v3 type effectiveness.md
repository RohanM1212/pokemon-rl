This is devlog entry 3, looking at reward function v3. This version is the most interesting to me because it forces the agent to learn something actual about the game world, instead of just balancing the raw math of HP bars. I added a type effectiveness bonus to the system. If the agent hits the enemy with a super effective move, it gets an extra +0.5 reward. If it picks a move that is not very effective, it takes a -0.3 penalty. The type chart is hardcoded directly from the game data, so water beats fire, electric beats water, and everything matches the actual game logic.
The hypothesis was that the agent would learn to prefer situational moves that match the enemy's weakness over moves that just have high base power.
This is where the project connects directly to robotics. A robot traveling through a warehouse does not just look at a map and pick the straightest path every single time. It needs to know if a specific corridor is usually blocked by obstacles at a certain hour, or if a surface type is too slick for its wheel grip. The AI agent learning to recognize type matchups uses the exact same cognitive structure as a physical robot learning to adapt to its physical surroundings.
What I found during the training run was that the agent actually started picking up on the context. In the earlier versions, it would just find one high power normal move and click it until the fight ended. With the v3 rewards, it started switching its attack choice depending on which enemy sprite was on screen. Because a super effective hit deals double damage anyway, adding the extra +0.5 reward made the mathematical incentive too big for the network to ignore. It learned that a lower power move with the right type alignment was worth more than a massive neutral attack.
The thing is, the agent is finally showing some real tactical behavior instead of just running a simple optimization loop on damage numbers. It is choosing actions based on the state of the environment rather than just pressing its favorite button. This proves that you can guide a model toward complex contextual choices without explicitly programming the decision tree yourself.
